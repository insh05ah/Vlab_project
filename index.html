PASSING ARGUMENTS TO A FUNCTION

Aim

The aim is to demonstrate the various ways arguments can be passed to a function, including positional arguments, keyword arguments and default arguments.

Theory

Arguments are the values passed inside the parenthesis of the function. A function can have any number of arguments separated by a comma.
There are different argument types available to pass values to functions, making them more flexible and reusable. The following are some function argument types in Python: Default arguments, Keyword arguments, Positional arguments.

Default Arguments
Default arguments are parameters in a function that are assigned predefined values in the function definition. These default values are used if the caller does not provide a corresponding argument during the function call. The default value is assigned by using the assignment(=) operator of the form keywordname=value.
Syntax: def function_name(var1, var2=def_value2, var3=def_value3) function_name(val1, val2)
Keyword Arguments
Keyword arguments in Python refer to arguments passed to a function using their parameter names during the function call. When using keyword arguments, the order of the arguments does not affect the output, as long as the logic of the code is correct. This ensures that the function works as intended regardless of the argument order.
Syntax: function_name(var1=def_value1, var2=def_value2, var3=def_value3)
Positional-Only Arguments
Positional arguments in Python are values passed to a function in a specific order, and the order matters. If the order of the arguments differs from the order of the parameters defined in the function, it may lead to unexpected output. Positional arguments further enforce that arguments must be passed in the defined order without using keywords. Positional arguments are best used when the order of the arguments is well-defined and easy to remember.
Syntax: function_name(value1, value2, value3)

This allows flexibility in the order of values. In both cases, the output will be correct, provided that the code logic is correct.



Positional arguments

Define a function and specify the parameters. Call the function with positional arguments: When calling the function, provide arguments in the order that matches the parameter sequence defined in the function.


Calling the function in the wrong order will result in an incorrect output.


Hence, the position of the arguments matter.
